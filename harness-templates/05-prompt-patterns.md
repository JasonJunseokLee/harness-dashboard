# 05. 프롬프트 엔지니어링 패턴 (쇼핑몰 데이터 수집 익스텐션 맞춤)

> 하네스 설계에서 발견된 패턴을 이 프로젝트의 DOM 파싱·데이터 추출·비개발자 UX 문맥에 맞게 구체화한 버전

---

## 핵심 발견: 워딩이 결과를 좌우한다

### 이 프로젝트에서의 "Museum Quality" 상당 표현

DOM 추출 정확도나 팝업 UI를 Generator에게 지시할 때 **"비개발자가 처음 보자마자 의미를 이해할 수 있는"** 표현을 사용하면:
- 추가 피드백 없이도 필드 레이블·상태 메시지의 가독성이 향상되는 경향
- "추출 완료" 버튼 같은 핵심 액션 요소가 시각적으로 더 부각됨

**권고 워딩 예시:**
- 낮은 품질 유발: "팝업에 추출 결과를 표시하세요"
- 높은 품질 유발: "MD나 가격 담당자가 Chrome 우상단에서 1초 안에 성공/실패를 판단할 수 있도록 결과를 표시하세요"

---

## 패턴 1: 기준 언어로 행동 유도

채점 기준을 Generator에게 미리 공유하면 그 기준에 맞게 생성한다.

### 이 프로젝트 적용 예시 — DOM 파서 Generator

```markdown
## DOM 파서 Generator에게 전달할 품질 기준

당신의 결과물은 아래 기준으로 평가됩니다:

1. **범용성**: 특정 쇼핑몰 클래스명을 하드코딩하지 않는다
   - 높은 점수: 쿠팡·네이버쇼핑·11번가 3개 사이트 동일 코드로 통과
   - 낮은 점수: `.coupang-price` 같은 사이트 전용 선택자 포함

2. **필드 완성도**: 추출 대상 필드 누락률 0%
   - 높은 점수: 상품명·가격·이미지·옵션·설명·리뷰 수 모두 반환
   - 낮은 점수: 옵션·리뷰 필드가 null 또는 빈 배열로 반환

3. **견고성**: 페이지 구조 변경에 Graceful Degrade
   - 높은 점수: 일부 필드 추출 실패 시 나머지 데이터는 정상 전송
   - 낮은 점수: 하나라도 실패 시 전체 추출 중단

이 기준을 인지하고 구현하세요. 최종 평가는 QA 에이전트가 수행합니다.
```

### 이 프로젝트 적용 예시 — 팝업 UI Generator

```markdown
## 팝업 UI Generator에게 전달할 품질 기준

1. **조작 효율성**: 핵심 액션(수집 시작·전송)은 팝업 열자마자 1탭 이내
2. **상태 가시성**: 수집 중·완료·실패 상태가 색상+텍스트 동시 표현
3. **비개발자 친화성**: 에러 메시지에 기술 용어(404, CORS 등) 미노출

이 기준을 참고하되 자기 평가는 하지 마세요.
```

## 패턴 2: Few-shot 캘리브레이션

Evaluator의 점수가 사이트·상황마다 드리프트하지 않도록 앵커링.

### DOM 추출 정확도 캘리브레이션 앵커

```markdown
## 추출 품질 평가 캘리브레이션

### 앵커 A — 9/10 (우수)
- 상품명·가격·대표이미지·모든 옵션 조합·상품설명·리뷰 수·별점 추출 성공
- 가격이 할인가/정가 모두 올바른 키에 매핑
- 추출 소요 시간 < 500ms
- 네이버쇼핑·쿠팡·11번가·지마켓 4개 사이트 모두 통과

### 앵커 B — 5/10 (보통)
- 상품명·가격·이미지는 추출되나 옵션 또는 리뷰 중 하나 누락
- 추출은 되나 가격 필드에 "₩" 문자열이 포함되어 파싱 후처리 필요
- 2~3개 사이트에서 정상, 나머지는 빈 값

### 앵커 C — 2/10 (미흡)
- 상품명만 추출되거나 페이지 전체 텍스트를 원시 덤프
- 이미지 URL이 썸네일 CDN URL이 아닌 data-URI 또는 상대경로
- 오류 없이 완료로 표시하지만 전송 데이터가 비어 있음
```

### 팝업 UX 캘리브레이션 앵커

```markdown
### 앵커 A — 9/10 (우수)
- 아이콘 클릭 → 수집 완료 → 전송까지 3초 이내, 진행 상태 실시간 표시
- 설정 화면에서 엔드포인트 변경 후 저장 시 즉시 반영 확인 메시지
- 오류 시 "서버 연결 실패, 엔드포인트 주소를 확인하세요" 수준의 안내

### 앵커 B — 5/10 (보통)
- 수집 중 상태가 표시되나 완료 후 몇 초간 UI 변화 없음
- 설정 저장 성공/실패 피드백 없음
- 에러 코드가 그대로 노출 ("Error: 401 Unauthorized")

### 앵커 C — 2/10 (미흡)
- 수집 버튼 클릭 후 아무 반응 없음 (콘솔에서만 확인 가능)
- 전송 실패해도 성공으로 표시
- 팝업 레이아웃이 Chrome 익스텐션 팝업 최대 너비(800px) 초과
```

## 패턴 3: 합리화 방지 프롬프트

QA 에이전트가 문제를 발견하고도 "일부 사이트에서는 작동하니 괜찮다"고 넘기는 것을 방지.

### 이 프로젝트에서 자주 발생하는 합리화 패턴

```
❌ 안티패턴
발견: 쿠팡에서 옵션 추출 실패
판단: 하지만 네이버쇼핑·11번가는 정상이므로 범용성 7/10

✅ 올바른 패턴
발견: 쿠팡에서 옵션 추출 실패
판단: 범용성 4/10. 국내 2위 쇼핑몰 미지원은 범용 파서의 핵심 결함.
수정 필요: 쿠팡 옵션 선택 DOM 구조(.prod-option__item) 또는 JSON-LD 활용 방안 명시
```

```
❌ 안티패턴
발견: 전송 실패 시 에러 메시지에 "CORS" 노출
판단: 하지만 개발자 모드 사용자는 이해할 수 있으므로 UX 8/10

✅ 올바른 패턴
발견: 비개발자 타겟 사용자에게 기술 용어 노출
판단: UX 5/10. 핵심 타겟(비개발자 데이터 운영자)이 스스로 해결 불가.
수정 필요: "서버로 데이터를 보내지 못했습니다. 엔드포인트 주소와 인증 토큰을 확인해 주세요."로 대체
```

### Evaluator 프롬프트 규칙

```markdown
## 평가 규칙 — 쇼핑몰 데이터 수집 익스텐션

- 발견한 문제는 사이트 이름과 필드명을 특정하여 기록하세요.
- "일부 쇼핑몰에서는 작동한다"는 이유로 범용성 점수를 올리지 마세요.
- 비개발자 사용 시나리오로 재현 가능한지 기준으로 판단하세요.
- 전송 성공/실패 판단은 UI 표시가 아닌 실제 서버 응답 기준으로 하세요.
- 임계값 미달 시: [사이트/기능 → 실제 발생 현상 → 기대 동작 → 수정 방향] 순으로 명시하세요.
```

## 패턴 4: 단계적 스펙 확장

Planner가 기능 요청을 완전한 구현 스펙으로 확장할 때 사용.

### 이 프로젝트 맞춤 확장 프롬프트

```markdown
아래 기능 요청을 Chrome Extension 구현 스펙으로 확장하세요.

## 요청
"[기능 요청 1~4문장]"

## 확장 규칙
1. Content Script / Background Service Worker / Popup 중 어느 레이어에서 처리할지 명시
2. Chrome Storage(sync vs local) 사용 여부와 저장 키 구조 포함
3. 비개발자 사용자가 설정하거나 확인할 수 있는 부분을 팝업 UI 명세로 포함
4. 쇼핑몰 DOM 구조 변경에 대한 Fallback 전략 명시
5. 빠진 요구사항·엣지케이스(로그인 필요 페이지, SPA 라우팅, 무한 스크롤 리뷰 등)를 P0/P1/P2로 표시
6. 구현 라이브러리·패턴 세부사항은 포함하지 않음

## 출력 형식
### 기능 요약
### 처리 레이어 (Content Script / Background / Popup)
### 데이터 흐름 (DOM 파싱 → 가공 → 전송)
### Chrome Storage 구조
### 팝업 UI 명세
### 엣지케이스 및 우선순위
```

## 패턴 5: 스프린트 계약 협상

Generator와 Evaluator가 각 스프린트 진입 전 범위를 합의.

### 이 프로젝트 Sprint 예시 (Sprint 2 — DOM 파서 v1)

```markdown
## Sprint 2 계약 협상

### Generator 선언
이번 스프린트에서 구현하겠습니다:
- 기능 A: 상품명·가격·대표이미지 추출 (필수 3필드)
- 기능 B: 쿠팡·네이버쇼핑 2개 사이트 검증
- 기능 C: 추출 실패 시 null 반환 (전체 중단 없음)

범위 밖: 옵션·리뷰 추출, 자동 전송, 팝업 UI 연동

### Evaluator 평가 기준 선언
- 기능 A 테스트: 각 필드가 실제 상품 페이지 텍스트와 일치하는지 확인
- 기능 B 테스트: 쿠팡·네이버쇼핑 각 5개 상품 URL로 추출 실행
- 통과 조건: 10건 중 8건 이상 3필드 모두 추출 성공, 실패 건은 에러 아닌 null ��환

양측 합의 → 스프린트 시작
```
