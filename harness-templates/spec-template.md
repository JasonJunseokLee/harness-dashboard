```markdown
# 제품 스펙 (Product Spec)

> Planner가 작성하는 문서. 사용자의 요청을 완전한 제품 설계로 확장한다.
> 이 문서가 승인되면 Generator의 작업 범위를 정의하는 **최상위 계약**이 된다.

---

## 1. 프로젝트 개요

### 한 줄 요약
클릭 한 번으로 모든 쇼핑몰 상품 페이지에서 상품 정보를 자동 추출하여 외부 API로 전송하는 Chrome Manifest V3 익스텐션.

### 배경 & 동기

**문제점**:
- 이커머스 분석가와 데이터 수집 플랫폼 운영자는 수백 개의 쇼핑몰에서 상품명, 가격, 이미지, 스펙, 리뷰를 정기적으로 수집해야 함
- 쇼핑몰마다 HTML 구조가 다르므로 각각 별도 크롤러를 개발해야 하는 높은 개발 비용 발생
- 사이트 구조 변경 시 크롤러 고장 → 유지보수 부담
- 비개발자 사용자는 스크래핑 도구를 직접 다루기 어려워 데이터 수집이 병목

**해결 방식**:
- 범용 DOM 파싱 알고리즘으로 특정 쇼핑몰에 종속되지 않음
- 사용자 경험: 상품 페이지 방문 → 익스텐션 아이콘 클릭 → 자동 추출 및 전송
- 별도 개발 없이 어떤 쇼핑몰에서도 즉시 동작

### 대상 사용자

| 페르소나 | 설명 | 핵심 니즈 | 기술 수준 |
|---------|------|---------|---------|
| **데이터 수집 플랫폼 운영자** (주) | 데이터 수집 SaaS를 운영하는 팀 리드/엔지니어 | 고객사 연동 간편화, 스크래핑 개발 비용 절감 | API 연동 가능, 설정 이해 가능 |
| **이커머스 MD / 가격 모니터링 담당자** (주) | B2B SaaS 고객사의 데이터 운영팀 | 코딩 없이 브라우저만으로 데이터 수집 완결 | 기본 설정 가능 (비개발자) |
| **데이터 분석가** (보) | 상품 데이터 기반 분석을 수행하는 조직 | 정확한 데이터 추출, 정기적 갱신 자동화 | API 개념 이해 |

### 성공 기준

- [x] 범용 DOM 파서가 5개 주요 쇼핑몰(G마켓, 쿠팡, 11번가, 옥션, 네이버 쇼핑)에서 기본 필드 추출 성공 (Sprint 2 완료)
- [x] 팝업 UI에서 필드 선택, 엔드포인트 설정, 즉시 전송 가능 (Sprint 1 완료)
- [x] 추출 미리보기 기능이 정확도 > 95% 달성 (Sprint 2 완료)
- [x] 인증 토큰 관리 및 암호화 저장 구현 (Sprint 1 완료)
- [x] E2E 테스트 커버리지 > 80% (Sprint 6 완료)
- [x] 크롬 웹스토어 게시 및 5개 쇼핑몰 공식 테스트 완료 (Sprint 6 완료)

## 3. 기술 아키텍처 (고수준)

### 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 익스텐션 규격 | Chrome Manifest V3 | 보안 강화, 크롬 정책 준수, 향후 규제 대응 |
| 익스텐션 로직 | TypeScript + Vanilla JS (Content Script) | 가벼움, 의존성 최소화, 빠른 페이지 응답 |
| 팝업 UI | React 18 + Vite | 모던 개발 경험, HMR, 빠른 번들 |
| 스타일링 | Tailwind CSS (shadcn/ui 옵션) | 빠른 프로토타이핑, 일관성 |
| 스토리지 | Chrome Storage API (sync 모드) | 크로스 디바이스 설정 동기화, 보안 |
| 빌드 도구 | Rollup + esbuild | 경량 번들, Chrome Extension 호환성 |
| 테스트 | Vitest (단위) + Playwright (E2E) | 빠른 피드백, 브라우저 자동화 |
| 타입 체크 | TypeScript + ESLint | 버그 조기 발견 |
| CI/CD | GitHub Actions | 자동 테스트, 자동 빌드 |

### 시스템 구조

```
┌─────────────────────────────────────────────────────┐
│          Chrome 브라우저 & 익스텐션 실행             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────┐  ┌─────────────┐  ┌───────────┐ │
│  │ Content Script│  │  Background │  │  Popup UI │ │
│  │  (TypeScript) │  │  Service    │  │ (React 18)│ │
│  │              │  │  Worker     │  │           │ │
│  │ ▸ DOM 파싱   │  │ ▸ 메시지    │  │ ▸ 설정    │ │
│  │ ▸ 데이터 추출│  │   라우팅   │  │ ▸ 미리보기│ │
│  │ ▸ 스크롤     │  │ ▸ 토큰 관리 │  │ ▸ 전송    │ │
│  └──────────────┘  └─────────────┘  └───────────┘ │
│       ↓                 ↓                  ↓        │
│  ┌───────────────────────────────────────────────┐ │
│  │   Chrome Storage API (로컬 암호화)             │ │
│  │   - 엔드포인트 URL                              │ │
│  │   - 인증 토큰                                   │ │
│  │   - 필드 선택 설정                              │ │
│  │   - 수집 이력                                   │ │
│  └───────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
                       ↓
            ┌──────────────────────┐
            │  사용자 설정 API     │
            │  (외부 엔드포인트)   │
            │                      │
            │ POST /api/collect    │
            │ {                    │
            │   product_name: "...",
            │   price: 29000,      │
            │   image_url: "...",  │
            │   ...                │
            │ }                    │
            └──────────────────────┘
```

### 외부 의존성

| 서비스/API | 용도 | 위험 요소 | 대안 |
|-----------|------|---------|------|
| Chrome Web Store | 익스텐션 배포 | 검수 거부, 계정 정지 | 사내 배포 (Enterprise) |
| 사용자의 데이터 수집 API | 추출 데이터 전송 | 엔드포인트 다운, 인증 실패 | 로컬 큐잉 + 재시도 로직 |
| npm 라이브러리 (React, Vite 등) | 개발 의존성 | 공급망 공격, 버전 호환 문제 | lock 파일 관리, 보안 감시 |

## 4. 스프린트 분해 계획

### 스프린트 로드맵

```
Sprint 0: 프로젝트 셋업 & 아키텍처 결정

    ↓

    ↓
Sprint 3: 미리보기 + 필드 선택 + 즉시 전송
    ↓

```
- **목표**: 개발 환경 완성, 기술 스택 확정, 리포지토리 초기화
- **포함 기능**: 없음 (인프라/프로세스만 구성)
- **완료 조건**:
  - [x] TypeScript + Rollup 빌드 파이프라인 완성
  - [x] React + Vite 팝업 개발 환경 구성
  - [x] Vitest + Playwright 테스트 환경 구성
  - [x] GitHub Actions CI/CD 파이프라인 설정
  - [x] Chrome Extension manifest.json 초기 작성
  - [x] 프로젝트 README & 기여 가이드 작성
- **의존성**: 없음
- **예상 QA 레벨**: Level 0 (테스트 불필요)
- **예상 기간**: 3~5일

#### Sprint 1: 팝업 UI + 익스텐션 기본 구조

- **포함 기능**: #2 (팝업 UI - 기본 화면), #5 (수집 필드 선택)
- **완료 조건**:
  - [x] 팝업 UI 레이아웃 구성 (React)
  - [x] 필드 선택 토글 (상품명, 가격, 이미지, 옵션, 설명, 리뷰)
  - [x] Content Script ↔ Background ↔ Popup 간 메시지 통신
  - [x] 팝업에서 설정 페이지로 이동 버튼
- **의존성**: Sprint 0 완료 필요
- **예상 QA 레벨**: Level 1 (단위 테스트 + 기본 통합 테스트)

#### Sprint 2: 범용 DOM 파서 + 데이터 추출
- **목표**: 5개 주요 쇼핑몰(G마켓, 쿠팡, 11번가, 옥션, 네이버 쇼핑)에서 상품 정보를 자동 추출
- **포함 기능**: #1 (범용 DOM 파싱)
- **완료 조건**:
  - [x] DOM 파싱 알고리즘 구현 (가격, 상품명, 이미지, 옵션 감지)
  - [x] 5개 쇼핑몰별 테스트 페이지에서 추출 성공률 > 90%
  - [x] 추출 데이터를 메모리에 임시 저장
  - [x] Playwright E2E 테스트 (각 쇼핑몰 1개 상품)
- **의존성**: Sprint 1 완료 필요
- **예상 QA 레벨**: Level 2 (단위 테스트 + E2E 테스트 + 실제 페이지 테스트)
- **예상 기간**: 7~10일

#### Sprint 3: 미리보기 + 필드 선택 + 즉시 전송
- **목표**: 추출된 데이터를 팝업에서 미리보기하고, 필터링 후 API로 전송
- **포함 기능**: #2 (팝업 UI 확장), #3 (즉시 전송), #5 (필드 선택 고도화), #6 (전송 상태 표시)

  - [x] "전송" 버튼 클릭 → 설정된 엔드포인트로 POST 요청
  - [x] 전송 상태 (진행 중, 성공, 실패) 팝업에 표시
  - [x] 네트워크 탭에서 요청 확인
- **의존성**: Sprint 2 완료 필요
- **예상 QA 레벨**: Level 2 (단위 테스트 + 통합 테스트 + 실제 API 요청 테스트)
- **예상 기간**: 5~7일

#### Sprint 4: 인증 토큰 관리 + 암호화 저장

- **완료 조건**:

  - [x] 토큰을 Chrome Storage API로 암호화 저장

## 9. 첨부: 주요 용어 정의

| 용어 | 정의 |
|------|------|
| **범용 DOM 파서** | HTML 구조를 분석하여 쇼핑몰 특정 규칙 없이 상품 정보를 추출하는 알고리즘 |
```
